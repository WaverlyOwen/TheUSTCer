# TheUSTCer

`TheUSTCer` 是一个以中国科大校园元素为主题的划线解谜游戏。玩家从左上角起点出发，画一条连续路径走到右下角出口；这条路径会把棋盘切成若干区域，而每个区域都必须同时满足题面里的约束。随着关卡推进，棋盘会变大、规则会叠加、题目密度会提高，目标是在经典模式里一路把 GPA 刷到 `4.30`。

项目现在已经不再是最初那种单一规则的小解谜，而是包含：

- 一个会持续爬升 GPA 的经典模式
- 一个从 `3×3` 起步的计时冲关模式
- 一个可自定义尺寸、题型和强度的挑战模式
- 一套带局部覆盖质控和路径变异修补的出题器
- 一套仅开发环境可见的 console 调试入口

---

## 1. 核心玩法

每一题都是一个网格。你需要：

1. 从左上角圆点出发
2. 画一条不自交的连续路径
3. 到达右下角出口
4. 让这条路径围出的所有区域都满足题面规则

如果答案不满足要求，游戏会把出错的格子和未穿过的黑色路名高亮出来，并播放失败反馈。

### 路径的本质

路径不是“经过哪些格子”，而是“沿格点之间的边走”。因此它的作用是把棋盘切割成多个连通区域，判题时检查的是这些区域是否满足约束。

## 2. 题目规则

当前版本主要有四类约束，它们可能单独出现，也可能叠加出现。

### 2.1 书院题

书院格会带颜色与院系简称。

规则是：

- 同一个区域里，不能出现多种不同颜色的书院
- 也就是说，一个区域里的书院必须“同院”

当前题库里的书院类型包括：

- 光启仲英书院（橙）
- 冲之书院（蓝）
- 时珍书院（绿）
- 守敬书院（紫）

### 2.2 组别题

有两组成对规则：

- `红 / 专`
- `理 / 实`

规则是：

- 同一个区域里，同组字只能各出现一次
- 如果区域里出现了 `红`，就必须同时出现 `专`
- 如果区域里出现了 `理`，就必须同时出现 `实`
- 只有一半、或者某个字重复，都会判错

### 2.3 路名题

题面中部分黑色边会标注校园路名。  
规则很直接：

- 每一条黑色路名边都必须被答案路径穿过

它不要求区域形状，只要求你的路径本身实际经过那条边。

### 2.4 字母题

字母题是当前版本最特殊的一类规则。题面里的字母格属于 `U / S / T / C` 四个家族之一，规则是：

- 字母所在的整个区域，必须与该字母形状**完全同构**
- 只允许**平移同构**
- **不允许旋转**
- **不允许镜像**

也就是说，判题时不是看“像不像”，而是把区域归一化以后，必须和预定义字模一模一样。

#### 当前字母规则的关键点

- 四个字母都使用固定朝向的 `4×5` 字模
- 字母优先要求只贴住棋盘的一条外边
- **不能同时贴住两条边**
- 只有在很小的棋盘上、某字母根本没有“单边贴靠”放置时，才允许临时豁免为贴两边甚至三边

这一点很重要，因为“贴两边”太容易把字母题做平。现在的设计目标是：能单边贴就单边贴，只有板子太小才放宽。

#### 字母题何时出现

字母题不再按固定关卡硬开，而是按**当前棋盘尺寸是否真的放得下四个字母家族**决定。

在当前尺寸曲线下：

- 经典模式前期不会出字母题
- 到 `7×7` 左右开始，四个字母才全部具备合法放置
- 也就是大约从 `level 60` 开始，字母题才会进入常规题池

#### 字母题的出题策略

生成器现在不会再从全部字母放置里胡乱抽样，而是先按“字母家族”均匀挑选，再找合法放置与路径候选。这样做的目的是：

- 降低以前 `U / C` 高频刷屏的问题
- 提高 `S / T` 的出现机会
- 在高压配置下支持双字母题
- 保持字母题的形状合法性和路径可解性

## 3. 三种游戏模式

游戏内通过左上角“模式”菜单切换三种模式。

### 3.1 经典模式

经典模式是主线模式，也是默认进入的模式。

特点：

- 从最早的小棋盘开始
- 每过一关，内部 `level` 增加
- `level` 会映射成更高的 GPA 和更大的棋盘
- 存档会自动保存
- 可随时在菜单里重置经典进度

### GPA 与棋盘尺寸

经典模式里，GPA 不是线性增长，而是按一条逐渐趋近 `4.30` 的曲线增长。  
棋盘尺寸则按关卡逐级变大，规则是：

- `size = [floor(level / 10 + 1.5), floor(level / 10 + 1)]`

所以你会看到：

- 最前期是很小的盘
- 每 10 级左右，尺寸会上一个台阶
- 到后期，棋盘会越来越宽、越来越高

### 经典模式的解锁内容

经典模式推进时会解锁额外玩法：

- GPA 到 `3.00`：解锁计时模式
- GPA 到 `3.70`：解锁挑战模式

游戏里到达这两个里程碑时会播放对应庆祝动画。

### 4.30 结局机制

经典模式首次把 GPA 推到 `4.30` 时：

- 会记录你首次达到 `4.30` 的用时
- 会播放 `4.30` 结局动画
- 会出现常驻的 `🎆` 重播按钮

但**经典模式不会在这里真正结束**。现在的逻辑是：

- `GPA` 显示会封顶在 `4.30`
- 你仍然可以继续刷题
- 每次通关后仍然会继续生成下一盘

也就是说，`4.30` 更像一个“通关里程碑”，不是“从此锁死不能玩”。

### 3.2 计时模式

计时模式用于“固定时长里能冲多少关”的玩法。

特点：

- 从 `3×3` 棋盘起步
- 可选时长：`10 / 20 / 30 / 45` 分钟
- 倒计时结束后会结算本轮成绩
- 不写入经典进度

计时模式里会显示：

- 剩余时间
- 已过关数
- 当前推进到的 GPA

如果时间到了，当前轮会结束，并显示本轮总过关数与最高 GPA。

### 3.3 挑战模式

挑战模式是自由配置入口，适合测生成器、刷特定类型题面、或者专门练某一类规则。

你可以在菜单里设置：

- 棋盘列数
- 棋盘行数
- 难度档位
- 是否启用字母题
- 是否启用书院题
- 是否启用组别题
- 是否启用路名题

### 尺寸范围

挑战模式的尺寸由滑块控制，范围固定为：

- `4 × 4` 到 `36 × 36`

这是一个经过生成耗时实测后定下来的范围。当前版本在上限附近仍然可以接受，只是会进入更明显的加载流程。

### 字母题开关限制

挑战模式里，字母题并不是所有尺寸都能开。

当前规则是：

- 当 `宽 × 高 > 22 × 22` 时，字母题会自动关闭
- 原因是超大盘上字母题的视觉意义会变弱，同时会明显拉高生成成本

### 难度档位

挑战模式目前有三档：

- `轻松`
- `标准`
- `高压`

难度不是简单改“题目更难”，而是会同时影响生成器的若干质量目标，例如：

- 局部覆盖密度
- 路径变异修补次数
- 路径最低覆盖率
- 是否更积极地产生更复杂的题面
- 是否允许更高概率出现双字母题

其中：

- `轻松` 更宽松，适合看机制
- `标准` 是默认体验
- `高压` 会更强调致密路径和复杂题面

## 4. 出题器与质控机制

当前版本的生成器不是单次随机游走后直接交卷，而是一个多阶段筛选系统。

大体流程是：

1. 先根据模式、尺寸、难度和题型开关决定生成参数
2. 生成若干候选路径
3. 对候选进行评分
4. 如果局部留白不好看，会做路径变异和重接
5. 在预算时间内挑选更优答案

### 当前质控方向

README 不去硬背所有内部常数，但整体目标是：

- 避免大块空白
- 避免区域分布过于失衡
- 避免早期字母题天然偏科
- 避免答案路径过短、过疏
- 保证题目与缓存答案自洽

### 局部覆盖

生成器现在会检查答案在局部窗口内的覆盖情况，避免棋盘上出现大片完全没被路径“照顾到”的留白区域。

同时还会做一轮“朝空白处修补”的变异流程：

- 找出较空的局部区域
- 选取靠近该区域的路径切点
- 重接后续路径
- 尝试把答案线重新引向那片空白

这就是为什么现在题目相比早期版本更“铺得开”，不那么容易出现一边挤满、一边大片发呆的盘。

### 字母题候选筛选

字母题的生成会额外考虑：

- 字母是否合法放置
- 双字母之间是否互不干扰
- 路径切出来的区域数是否过少
- 是否存在明显巨型空区

如果字母题候选质量不够，生成器会回退到：

1. 双字母失败回退为单字母
2. 单字母失败再回退到普通题

所以高规则强度不等于硬塞烂题。

## 5. 操作方式

### 5.1 桌面端

- 鼠标拖动画线
- 反向回拖可以撤销
- `W / A / S / D` 或方向键：移动
- `Backspace`：撤销一步
- `R`：清空当前路径
- `Enter`：提交答案
- `Tab`：换题
- `Control`：显示答案
- `Shift`：隐藏答案

### 换题与看答案的代价

当前版本里：

- `Tab` 换题会降低当前局面的难度推进
- `Control` 看答案也会降低当前局面的难度推进

经典模式下，这相当于会把内部 level 往回扣一格；计时模式下也会影响当前轮的推进等级。  
所以它们是正式存在的“提示成本”，不是纯展示功能。

### 5.2 移动端

移动端使用：

- 滑动手势划线
- 底部按钮执行 `清空路径 / 提交答案 / 显示答案 / 隐藏答案 / 更换地图`

右上角 `?` 会打开帮助卡片。

当前帮助内容包括：

- 不可旋转
- 不可镜像
- 优先单边贴靠

## 7. 存档与进度

游戏默认在浏览器本地保存这些内容：

- 经典模式进度
- 首次到达 `4.30` 的用时
- 模式解锁状态
- 计时模式默认时长
- 挑战模式的最近配置
- 是否看过 `4.30` 结局

### 存档原理

当前实现使用浏览器本地存储（`localStorage`）保存这些状态，所以：

- 同一浏览器里刷新页面，进度会保留
- 切换设备或清空浏览器数据后，进度不会自动同步
- 某些应用内浏览器如果禁用了或隔离了本地存储，保存效果可能不稳定

另外，菜单、加载、庆祝动画、结局动画这些“非游玩时段”不会计入经典模式用时，也不会继续消耗计时模式倒计时。

## 8. 开发调试

### 8.1 URL 调试入口

开发时可以通过 URL 参数直接从指定关卡启动：

```text
?level=N
```

例如：

```text
http://localhost:5173/?level=60
```

这类会话不会写入正式经典存档，适合本地测题。

### 8.2 Dev-only Console API

在 `vite` 开发环境下，页面会额外挂一个只供调试使用的全局对象：

```js
window.__ustcerDebug
```

可用命令包括：

- `help()`：打印可用调试命令
- `state()`：查看当前关卡、尺寸、题面字母等状态
- `setLevel(level)`：切到指定关卡并重开当前题
- `reload()`：按当前级别重抽一题
- `solve()`：直接灌入缓存答案并正常提交
- `showAnswer()` / `hideAnswer()`：显示或隐藏答案线
- `seekLetter('U' | 'S' | 'T' | 'C')`：循环重抽，直到出现指定字母
- `seekLetters(['U', 'C'])`：循环重抽，直到出现指定字母组合
- `playEnding()`：直接播放 `4.30` 结局
- `resetProgress()`：清空本地经典进度并重开

快速入口：

```js
window.__ustcerDebug.help()
```

注意：

- 这些接口只在开发环境可见
- 正式构建不会暴露 `__ustcerDebug`
- 旧的底层入口 `window.__ustcer` 仍保留作兼容用途

## 9. 本地开发

需要 Node.js `>= 18`。

```bash
npm install
npm run dev
npm run build
npm test
npm run preview
```

命令说明：

- `npm run dev`：启动开发服务器
- `npm run build`：构建到 `dist/`
- `npm test`：运行逻辑与调试接口测试
- `npm run preview`：本地预览构建产物

项目可以直接部署到 GitHub Pages。

推荐方式：

## 12. 当前版本要点

如果你是从旧版本 README 过来的，当前机制里最值得记住的变化是：

- 字母题使用固定朝向 `4×5` 字模
- 字母不能旋转、不能镜像
- 字母优先只贴一条边，只有小盘放不下时才豁免
- 字母题按“当前尺寸是否放得下四家族”解锁，不再按旧的固定低关卡出现
- 经典模式到 `4.30` 后不会锁死，仍可继续玩
- 计时模式现在从 `3×3` 开始
- 挑战模式尺寸改为滑块，范围固定为 `4 - 36`
- 挑战模式大盘会自动关闭字母题
- 生成器加入了更积极的局部覆盖质控和路径变异修补

这版 README 的目标就是让文档终于跟游戏本体站在同一个宇宙里。
