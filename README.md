# TheUSTCer

`TheUSTCer` 是一个以中国科大校园元素为主题的划线解谜游戏。从左上角起点画一条连续路径走到右下角出口，路径会把棋盘切成若干区域，每个区域都必须满足题面约束。随关卡推进，棋盘变大、规则叠加、题目密度提高，目标是在经典模式里把 GPA 刷到 `4.30`。

当前版本包含：

- **经典模式**：GPA 从 `1.00` 一路爬到 `4.30` 的主线
- **计时模式**：固定时长内连续冲关（GPA `3.00` 解锁）
- **挑战模式**：自定义尺寸、题型与难度（GPA `3.70` 解锁）
- **题目工坊**：收藏生成题、创作自定义题（GPA `4.00` 解锁创作）、把整道题编码进链接分享
- **章节式教程**：随游玩进度逐步解锁，示例板由游戏渲染器实时绘制
- **深浅双主题**：跟随系统 + 手动三态切换，全界面（含题板 SVG）适配
- **全端交互**：画线到出口自动判定；单指画线、双指缩放平移；桌面滚轮矢量缩放
- **无尽模式**：每题限时、逐题收紧的生存挑战（GPA `3.30` 解锁）
- 一套带局部覆盖质控与路径变异修补的出题器，以及仅开发环境可见的调试入口

---

## 1. 核心玩法

每一题都是一个网格：

1. 从左上角圆点出发
2. 画一条不自交的连续路径
3. 走到右下角出口——**到达出口的瞬间自动判定**，无需手动提交
4. 让路径围出的所有区域都满足题面规则

判错时会保留路径：出错的格子反色闪烁、漏掉的路名闪红，回拖修改后再次踏入出口会重新判定。

路径的本质不是"经过哪些格子"，而是"沿格点之间的边走"——它把棋盘切割成多个连通区域，判题检查的是这些区域是否满足约束。

## 2. 题目规则

四类约束可能单独出现，也可能叠加。

### 2.1 书院题

书院格带颜色与院系简称，**同一区域内只能出现一种颜色的书院**。颜色与书院的对应关系：

| 颜色 | 书院 | 院系字 |
| --- | --- | --- |
| 橙 | 光启·仲英书院（少年班学院） | 少 |
| 蓝 | 冲之书院（数学 / 工程 / 管理） | 管 工 数 |
| 绿 | 时珍书院（生物 / 信息 / 计算机） | 网 微 计 生 信 |
| 紫 | 守敬书院（物理 / 化学等） | 环 核 地 化 物 |

同色书院的不同学院可以同区（比如管院和工院都属冲之）。

### 2.2 组别题

对应校训「红专并进，理实交融」：

- 一个区域中，有 `红` 就必须有 `专`，有 `理` 就必须有 `实`
- 一个区域内不能出现多个红专对或多个理实对

### 2.3 路名题

所有黑色标出的路名边（如孺子牛路），都必须被路径穿过。

### 2.4 教学楼题

出现教学楼块（汉字数字 `一二三四五`，格内以淡灰小方块拼出楼形）时，它所在的区域必须与该楼形状**完全一致**：

| 楼 | 形状 | 由来 |
| --- | --- | --- |
| 一教 | 凹 | 物理实验教学中心 |
| 二教 | 凸 | 临接水上报告厅 |
| 三教 | Z | 非垂直结构的抽象 |
| 四教 | 方 | 在线教育 App 图标 |
| 五教 | 线 | 东区北门的直线设计 |

- **允许旋转与镜像**（与旧字母题不同），五栋楼都能装进 `2×3`
- 多栋不同的楼可同图出现，但同一栋不重复，且各占独立区域（不做跨区域拼砌）
- 从 `2×3` 小盘起低概率出现，关卡越深概率与栋数越高

## 3. 游戏模式

左上角 `☰` 菜单切换模式，菜单里同时住着题目工坊和设置。

### 3.1 经典模式

主线模式，进度自动存档，可随时重置。

- GPA 按前快后缓的有界曲线增长，第 `80` 关整达到 `4.30`；满绩后每再通 10 关记 1 学分（`×n Credit` 小字）
- 棋盘尺寸 `[floor(level/10 + 1.5), floor(level/10 + 1)]`，每约 10 级上一个台阶
- 查看答案 / 更换题目会扣一级关卡推进——是正式的"提示成本"

解锁里程碑（达成时播放庆祝动画）：

| GPA | 解锁 |
| --- | --- |
| `3.00` | 计时模式 |
| `3.30` | 无尽模式 |
| `3.70` | 挑战模式 |
| `4.00` | 题目工坊的创作功能（收藏 / 游玩 / 分享不受此限制） |
| `4.30` | 结局动画 + 常驻重播按钮；GPA 显示封顶，但可以继续刷题 |

首次到达 `4.30` 会记录用时；菜单、加载、庆祝动画等"非游玩时段"不计入用时。

### 3.2 计时模式

固定时长内能冲多少关：

- 可选 `10 / 20 / 30 / 45` 分钟，从 `3×3` 棋盘起步
- 每过一关难度升一级；看答案 / 换题同样扣级
- 时间到自动结算本轮成绩，不写入经典进度

### 3.3 无尽模式

每题独立倒计时，超时即本轮结束、结算最佳成绩：

- 棋盘从 `3×3` 起步逐步长到 `16×16`，之后每题随机 `8–16`
- 时限 ≈ `12s + 0.9s/格`（封顶 4 分钟），爬满后逐题 ×0.99 收紧，保底 `0.45s/格`
- 换题扣一半剩余时间；看答案剩余压到 10 秒

### 3.4 挑战模式

自由配置入口：棋盘 `4×4` 到 `36×36`（滑块）、三档难度（轻松 / 标准 / 高压）、四类题型开关。

- 难度档位同时影响生成器的多个质量目标（局部覆盖密度、变异修补强度、教学楼栋数等）
- 不影响经典进度

## 4. 题目工坊

`☰` 菜单里的题目工坊管理三种来源的题：收藏的生成题、自己创作的题、导入的分享题。

### 收藏与游玩

游玩任意生成题时点顶栏书签按钮即可收藏。工坊列表里可随时游玩 / 分享 / 删除，游玩自定义题走独立会话，不动三个正式模式的进度。

### 创作（GPA `4.00` 解锁）

编辑器支持：

- 新建 `2×2` 到 `20×20` 空白棋盘
- 点击格子设置类型：空白 / 灰底 / 四书院色 / 红专 / 理实 / 教学楼 / **自定义颜色**（取色器，进入选择列表；文字可选表、自定义或随机）
- 设置每条邻边的路名（选表 / 自定义 / 随机）
- 「画答案」模式：用正常画线操作录一条参考答案，也可以不录
- 创作题不保证可解；游玩无答案题时点 💡 会提示 `No Answer`

自定义颜色与书院色按**实际颜色**统一判冲突——给书院色配自定义字，判题仍算同一书院。

### 链接分享

分享会把整道题（题面、答案、自定义色表、路名表）用 base64url 编进 `#p=` 链接，打开链接确认后即导入游玩，无需账号或服务器。数据格式见 [docs/reference.txt](docs/reference.txt)。

## 5. 教程

右上角 `?` 打开章节式教程，**章节随游玩进度解锁**：

| 章节 | 解锁条件 |
| --- | --- |
| 基本规则 | 始终开放 |
| 书院规则 | GPA `1.50` |
| 教学楼 | GPA `2.00` |
| 红专与理实 | GPA `2.30` |
| 计时 / 无尽 / 挑战模式 | 对应模式解锁后 |
| 题目工坊 | 始终开放 |

规则示例板不是贴图，而是由游戏自己的 `BoardView` 实时渲染（自动跟随深浅主题），违规示例用红线标出。滑片支持指针拖动、按钮和方向键翻页。

## 6. 操作与界面

### 画线

- **移动端**：单指滑动画线；双指捏合缩放、平移棋盘
- **桌面端**：`W/A/S/D` 或方向键；或按住线头拖动（反向回拖撤销）；滚轮以指针为中心缩放
- `Backspace` 撤销一步，`R` 清空；`Enter` 手动判定（冗余入口，走到出口本来就会自动判定）
- 缩放采用**矢量方式**（直接改 SVG 尺寸而非位图变换），任意倍率清晰；放大时棋盘收进半透明圆角卡片

### 顶栏

左侧 `☰` 菜单；右侧依次：清空路径、显示答案（扣级）、更换题目（扣级）、收藏本题、教程。

### 设置（`☰` 菜单内）

- **外观**：跟随系统 / 浅色 / 深色，手动选择会记忆；题板 SVG、背景粒子、全部面板同步适配
- **画线灵敏度**：`0.5 – 1.5` 滑杆，同时调节移动端滑动触发距离与桌面拖动提交阈值，实时生效

## 7. 出题器与质控

生成器是多阶段筛选系统而非单次随机游走：

1. 按模式、尺寸、难度、题型开关决定生成参数
2. 生成候选路径并评分
3. 局部留白不佳时做路径变异与重接（找空白区 → 选临近切点 → 把答案线引向空白）
4. 在预算时间内挑选最优

质控目标：避免大块空白与区域失衡、避免答案路径过短过疏、保证题面与缓存答案自洽。教学楼题额外检查放置合法性、多楼互不干扰、区域数量与巨型空区，质量不够时按"多楼 → 单楼 → 普通题"回退。

---

## 8. 存档

全部状态存在浏览器 `localStorage`（前缀 `ustcer.`）：经典进度与用时、模式解锁、计时时长、挑战配置、结局标记、主题偏好、灵敏度、工坊题库。同浏览器刷新保留；换设备或清数据不同步；隐私模式下静默降级为不持久化。

## 9. 开发

需要 Node.js `>= 18`。

```bash
npm install
npm run dev      # 开发服务器（默认 5173，可用 PORT 覆盖）
npm test         # node --test 逻辑测试
npm run build    # 构建到 dist/
npm run preview  # 预览构建产物
```

### URL 调试入口

`?level=N` 从指定关卡启动（不写入正式存档），如 `http://localhost:5173/?level=60`。

### Dev-only Console API

开发环境挂载 `window.__ustcerDebug`（正式构建不暴露）：

- `help()` / `state()`：命令列表 / 当前状态
- `setLevel(n)` / `reload()`：切关卡 / 重抽当前题
- `solve()`：灌入缓存答案并判定
- `showAnswer()` / `hideAnswer()`
- `seekBuilding('三')` / `seekBuildings(['一','五'])`：重抽直到出现指定教学楼
- `playEnding()` / `resetProgress()`

## 11. 代码结构

```text
src/
  core/      路径模型、出题器、判题器、教学楼规则(buildings)、游戏状态、题目序列化(puzzle-io)
  render/    SVG 棋盘(board)、路径动画(animator)、缩放视口(viewport)
  input/     键盘、指针画线、触摸滑动、缩放平移手势(gesture)
  ui/        HUD、顶栏、模式菜单、教程、题目编辑器、toast、背景、结局动画
  lib/       存储、主题、设置、剪贴板、随机、HTML 转义
  debug/     开发环境调试接口
  styles/    主样式、教程、背景、结局
test/        node --test 逻辑测试
docs/        格子/路名编码与分享数据格式（reference.txt）
```

关键入口：

- 题面编码与分享格式：[docs/reference.txt](docs/reference.txt)、[src/core/puzzle-io.js](src/core/puzzle-io.js)
- 出题器：[src/core/generator.js](src/core/generator.js)；教学楼规则：[src/core/buildings.js](src/core/buildings.js)
- 判题：[src/core/validator.js](src/core/validator.js)；路径模型：[src/core/path.js](src/core/path.js)
